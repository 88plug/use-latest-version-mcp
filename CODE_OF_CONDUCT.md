# Contributor Code of Conduct

## Our Pledge

We commit to making participation in `use-latest-version-mcp` a harassment-free experience for everyone, regardless of background, identity, experience level, or opinion.

We expect contributors and maintainers to be direct, respectful, and focused on the work.

## Expected behavior

Examples of positive behavior:

- Be clear and factual.
- Critique ideas, not people.
- Assume good intent until evidence says otherwise.
- Keep feedback specific and actionable.
- Respect maintainers' time and the project's scope.

## Unacceptable behavior

Examples of unacceptable behavior:

- Harassment, insults, or personal attacks.
- Discriminatory language or behavior.
- Deliberate disruption of issues, reviews, or discussions.
- Repeated bad-faith argumentation.
- Publishing private security details publicly before coordination.

## Enforcement

Maintainers may remove, close, or lock content that violates this code of conduct.
They may also decline contributions that repeatedly ignore project boundaries or review feedback.

## Scope

This code of conduct applies to all project spaces, including issues, pull requests, review comments, and any community channels maintained for the project.

## Reporting

If you need to report behavior that violates this code of conduct, contact the maintainers privately.

Please include:

- What happened
- Where it happened
- When it happened
- Any supporting context

## Attribution

This code of conduct is based on common open source community standards and is intended to be practical rather than ceremonial.
